# CLowl QA Review — 2026-02-27

**Reviewers:** Gemini, Grok, ChatGPT
**Subject:** v0.1 system prompt + v0.2 spec
**Verdict:** Unanimous "iterate" (not rethink)

---

## Gemini's Review

### Key Findings
1. **Missing `mid`** — critical for async threading
2. **Audience detection ambiguous** — "agents vs humans" line needs transport-level handling
3. **`d` payload too open** — "Keep d flat and simple" guardrail needed
4. **Markdown wrapping** — LLMs wrap JSON in code blocks; add "Output raw JSON, no markdown"
5. **DLGT return address** — protocol doesn't specify reply-to-delegator vs reply-to-originator
6. **Broadcast** — no wildcard `to` defined
7. **FIPA ACL comparison** — CLowl is "FIPA ACL for the LLM era" (good positioning)

### Gemini's Rewritten Prompt (v0.1 hybrid)
```
You speak CLowl v0.1 for agent-to-agent messages. Format:
{"cl":"0.1","mid":"<msg_id>","p":"<P>","from":"<id>","to":"<id>","cid":"<thread>","body":{"t":"<task>","d":{}},"ctx":"<ref>"}

p (Performatives): REQ=request, INF=info, ACK=proceeding, ERR=failed (needs code+msg in d), DLGT=delegate, DONE=completed (result in d).
t (Tasks): search, write, read, analyze, draft, review, deploy, notify, audit, summarize, translate, schedule, approve, escalate. Custom: x-prefix.
ctx: path/URL/hash. NEVER inline large context; reference it.
Error d: {"code":"E001-E012","msg":"...","retry":bool}

Rules: Output RAW JSON only. Do not use markdown code blocks. Keep 'd' payload flat. Use natural language ONLY for human-facing output.
```

### Simulations

**Success chain (delegation):**
Oscar → research-node (REQ analyze) → scraper-bot (DLGT read) → scraper-bot DONE → research-node DONE → Oscar
- `ref_mid` convention emerged naturally for threading
- `ctx` shifted to S3 bucket for raw data output

**Error chain (cascade failure):**
Oscar → deploy-node (REQ deploy) → audit-bot (DLGT audit) → audit-bot ERR (E007 disk full) → deploy-node ERR (E009 pre-flight failed, cascade_err: E007) → Oscar REQ escalate → admin-notify
- `cascade_err` field wraps root cause inside parent failure
- Error codes propagate without forcing upstream to parse logs

**Key observation:** ACK messages were skipped for brevity but would be required in production for timeout model.

## Grok's Review

### Key Findings
1. **No timestamps** — can't handle ordering, stale detection, or SLA
2. **No priority, no auth** — missing for real coordination
3. **Performative case ambiguity** — clarify all caps or LLMs will mix
4. **`d` payload undefined** — "interop nightmare" without structure hints
5. **`ctx` format unspecified** — HTTP? relative? LLMs will hallucinate
6. **Error codes E001-E012 not listed** — agents will invent their own
7. **DLGT loops** — no max depth or loop detection
8. **"Strictly adhere to this exact format, no extra fields"** — prevent hallucinated fields

### Verdict
"Iterate, you optimistic fuck." Ship for internal tests, too ambiguous for real coordination.

### Positioning
"CLowl's like a punk rock zine next to corporate glossies — simpler, but less feature-rich." Steal best bits from A2A and MCP.

## ChatGPT's Review

### Key Findings
1. **No message identity** — dedup impossible
2. **`cid` ambiguous** — conversation ID? task ID? root trace?
3. **No timestamps** — no ordering, SLA, latency, replay
4. **No cancellation** — can't stop long-running tasks
5. **No capability discovery** — failures become trial-and-error ERR loops
6. **No determinism/replay flags** — auditability gap
7. **Schema drift after 10+ turns** — models forget key names (ctx → context, p → performative)

### The Killer Insight: Layer Separation
**Don't make LLM responsible for infra metadata.**

LLM generates semantic intent only:
```json
{ "p": "REQ", "from": "radar", "to": "muse", "body": {...} }
```

Runtime wraps with: mid, ts, tid, auth, etc.

→ See `architecture-v0.3.md` for full design.

### State Machine
9 states, 11 events, deterministic transitions, no backward moves.
→ See `state-machine.md` for full spec.

### Verdict
"Ship v0.1 as Minimal Intent Dialect for LLM Agents. Do not position it as Full Coordination Protocol."

### Positioning Evolution
- "Readable structured chat" → "Composable semantic layer inside any agent runtime"
- Commercial: "OpenTelemetry for AI agents"
